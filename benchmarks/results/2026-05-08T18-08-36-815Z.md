## Notes

- `cache-manager` v6+ uses Keyv stores under the hood, so Redis/SQLite/Mongo/MySQL/Postgres results for cache-manager and keyv are expected to be very close, with cache-manager's overhead reflecting only the wrapping layer.
- `delete` benchmarks reuse a rotating key pool of 1000 keys; once exhausted, subsequent calls measure no-op delete throughput. We re-populate the pool before each cycle, but Benchmark.js may run many iterations per cycle — the absolute number reflects the average across populated and exhausted states. Relative comparison between libraries is still valid.
- `set` benchmarks similarly rotate through the key pool, so the workload is set-or-overwrite.
- Asterisks (`*`) on a cell mean the library does not natively implement the operation and was emulated; for example, keyv has no native `setMany`, so it is implemented as `Promise.all` of singles.
